### Constitution-Aligned Requirements *(mandatory for this project)*

- **DAR-001 Data Accuracy**: Define event identity, deduplication keys, missing-event handling,
  late-arriving event behavior, and reconciliation rules for all behavior and metric data.
- **DAR-002 Traceability**: Define how user, visitor, session, product, order, payment,
  channel, and timestamp data connect the complete behavior path from visit to payment.
- **DAR-003 Permission Scope**: Define roles, data scopes, allowed operations, and forbidden
  access paths for each affected report, dashboard, export, or management function.
- **DAR-004 Privacy and Audit**: Define sensitive fields, masking rules, audit log events,
  and any authorized exceptions for viewing unmasked data.
- **DAR-005 Real-Time and Validation**: Define refresh expectations for core indicators and
  validation rules that prevent negative counts/amounts and out-of-range rates.
- **DAR-006 Analysis Usability**: Define required filters, chart types, visible metric units,
  freshness display, and empty/loading/error states.
