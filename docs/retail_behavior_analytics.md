# 智能零售用户行为分析系统文档总览

## 项目定位

本项目定位为一套前后端分离的智能零售用户行为分析系统。系统包含用户购物端和运营管理端，用户端负责产生真实业务行为，运营管理端负责行为采集管理、指标分析、转化漏斗、用户画像、智能推荐、报表和审计管理。

当前项目继续采用现有工程基础推进：

- 前端：React + TypeScript + Vite
- 后端：FastAPI + Pydantic + SQLAlchemy
- 数据库：MySQL
- 迁移工具：Alembic
- 图表：ECharts 或前端轻量图表组件
- 缓存：Redis 作为可选增强，用于热点指标缓存、刷新状态和任务锁

## 技术实现原则

1. 数据以 MySQL 为准，前端不作为业务数据源。
2. 用户行为、商品、用户、订单、支付、审计日志必须可持久化。
3. 前端按登录身份展示不同工作台，后端接口也必须校验角色权限。
4. 核心指标、漏斗、画像、推荐由后端基于 MySQL 数据计算。
5. 当前阶段不实现 HTTPS、bcrypt、生产级安全加固、千万级压测和 MySQL 分区优化。

## 用户与角色

内部管理端规划 5 类角色：

- 系统管理员：系统配置、角色权限、审计日志、数据重置、全量数据查看。
- 运营经理：运营总览、行为分析、漏斗、商品热度、预警、报表。
- 数据分析师：指标分析、漏斗分析、画像分析、推荐解释、报表。
- 客服主管：用户行为路径、用户画像、风险预警。
- 只读查看员：脱敏后的汇总看板、漏斗、商品排行、报表查看。

用户购物端规划至少 3 个演示用户：

- 高价值用户：购买频次高或支付金额高。
- 价格敏感用户：偏好低价商品，关注食品、家居、母婴等品类。
- 潜力用户：浏览、收藏、加购活跃，但支付较少。

## 核心业务模块

- 用户行为采集管理：浏览、点击、搜索、收藏、加购、下单、支付。
- 用户行为分析：访问量、点击量、浏览时长、来源渠道、搜索关键词。
- 转化漏斗分析：浏览到点击、点击到加购、加购到下单、下单到支付。
- 用户画像管理：基础属性、消费能力、兴趣偏好、活跃度、价值等级。
- 用户分群管理：高价值、潜力、新用户、价格敏感、沉默、流失风险。
- 商品热度分析：浏览、点击、收藏、加购、购买、转化率排行。
- 用户预警管理：低活跃、流失风险、转化异常、行为数据异常。
- 智能推荐分析：基于浏览、购买、相似用户、画像的推荐。
- 购买意向分析：高、中、低购买意向和加购未支付识别。
- 报表统计：行为明细、商品分析、漏斗、画像、分群、销售转化、运营效果。
- 安全审计：权限校验、敏感信息脱敏、操作日志。

## 当前与后续建设重点

当前页面和接口已经具备演示能力，但正式系统建设重点应调整为：

1. 将内存行为仓库替换为 MySQL 持久化仓库。
2. 将商品、用户、订单、支付、审计日志全部落库。
3. 将画像、推荐、漏斗、商品热度改为后端基于 MySQL 计算。
4. 完善 RBAC 后端权限校验。
5. 补充报表导出任务和导出文件管理。
