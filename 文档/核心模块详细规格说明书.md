# 核心模块详细规格说明书

## 1. 用户行为采集模块

### 1.1 模块目标

准确采集用户在购物端产生的关键行为，保证行为数据不重复、不漏记，并形成后续分析的数据基础。

### 1.2 行为类型

| 行为类型 | 英文字段 | 说明 |
|---|---|---|
| 浏览 | `browse` | 用户进入商品列表或商品详情 |
| 点击 | `click` | 用户点击商品、推荐位、按钮或活动位 |
| 搜索 | `search` | 用户输入关键词搜索商品 |
| 收藏 | `favorite` | 用户收藏商品 |
| 加购 | `cart_add` | 用户将商品加入购物车 |
| 下单 | `order_submit` | 用户提交订单 |
| 支付 | `payment_success` | 用户完成支付 |

### 1.3 采集字段

| 字段 | 说明 |
|---|---|
| `event_id` | 前端生成的全局唯一事件 ID |
| `source_system` | 来源系统，例如 customer-web |
| `event_type` | 行为类型 |
| `user_id` | 用户 ID |
| `visitor_id` | 访客 ID |
| `session_id` | 会话 ID |
| `product_id` | 商品 ID |
| `order_id` | 订单 ID |
| `payment_id` | 支付 ID |
| `channel_id` | 来源渠道 |
| `search_keyword` | 搜索关键词 |
| `occurred_at` | 行为发生时间 |
| `metadata` | 扩展信息，如商品名、价格、类目、品牌 |

### 1.4 去重规则

第一阶段采用双重规则：

1. `source_system + event_id` 唯一，防止重复上报。
2. 同一 `user_id/session_id + event_type + product_id` 在 1 秒内重复出现时视为重复行为。

### 1.5 输出结果

行为上报接口返回：

- `acceptedCount`
- `duplicateCount`
- `invalidCount`
- `delayedCount`
- `quarantinedCount`
- 每条事件的处理状态

## 2. 用户行为路径追踪模块

### 2.1 模块目标

按用户维度还原访问、浏览、点击、加购、下单、支付的完整路径。

### 2.2 路径阶段

```text
访问 / 浏览 -> 点击 -> 收藏 / 加购 -> 下单 -> 支付
```

### 2.3 查询能力

- 按用户 ID 查询行为时间线。
- 按会话 ID 查询单次访问路径。
- 按商品 ID 查询用户行为链路。
- 标记加购未支付、下单未支付等流失节点。

### 2.4 输出字段

- 行为时间
- 行为类型
- 商品
- 渠道
- 订单
- 支付
- 幂等状态
- 上一步行为来源

## 3. 实时指标看板模块

### 3.1 模块目标

为运营人员提供实时业务状态，包括访问、点击、加购、订单、销售和转化。

### 3.2 核心指标

| 指标 | 计算方式 |
|---|---|
| 访问量 | `browse` 事件数 |
| 点击量 | `click` 事件数 |
| 加购量 | `cart_add` 事件数 |
| 订单量 | `order_submit` 事件数或订单表数量 |
| 支付量 | `payment_success` 事件数 |
| 销售额 | 已支付订单金额合计 |
| 访问支付转化率 | 支付用户数 / 浏览用户数 × 100% |
| 下单支付转化率 | 支付订单数 / 下单订单数 × 100% |

### 3.3 数据刷新

当前阶段采用前端 5 秒轮询刷新。后续可接入 Redis 缓存和 WebSocket 推送。

### 3.4 校验规则

- 数量和金额不得为负。
- 转化率、流失率必须在 0 到 100 之间。
- 空数据时转化率和流失率显示为 0。

## 4. 转化漏斗分析模块

### 4.1 模块目标

分析用户从浏览到支付各阶段的转化情况，帮助定位流失环节。

### 4.2 漏斗阶段

| 阶段 | 事件 |
|---|---|
| 浏览 | `browse` |
| 点击 | `click` |
| 加购 | `cart_add` |
| 下单 | `order_submit` |
| 支付 | `payment_success` |

### 4.3 计算规则

- 每个阶段按用户或会话去重统计。
- 阶段转化率 = 下一阶段人数 / 当前阶段人数 × 100%。
- 阶段流失率 = 1 - 阶段转化率。
- 最大流失环节取流失人数最高的阶段。

## 5. 用户画像模块

### 5.1 模块目标

基于用户长期行为和订单数据识别用户偏好、消费能力、活跃度和价值等级。

### 5.2 画像维度

| 画像维度 | 计算依据 |
|---|---|
| 基础属性 | 用户表中的城市、会员等级、注册时间 |
| 兴趣偏好 | 浏览、点击、收藏最多的类目和品牌 |
| 消费能力 | 支付金额、客单价、购买频次 |
| 活跃度 | 最近访问时间、近 7 天行为次数 |
| 价值等级 | 支付金额和支付次数 |
| 价格偏好 | 浏览和购买商品的价格区间 |

### 5.3 分群规则

| 分群 | 规则 |
|---|---|
| 高价值用户 | 支付金额 ≥ 1000 或支付次数 ≥ 3 |
| 潜力用户 | 收藏 + 加购 ≥ 3 且支付次数 = 0 |
| 新用户 | 有访问行为但行为总量较低 |
| 价格敏感用户 | 平均购买价或浏览价位偏低 |
| 沉默用户 | 长时间无行为 |
| 流失风险用户 | 曾经活跃但最近一段时间无访问 |

### 5.4 展示方式

- 用户画像表格。
- 浏览类目偏好饼图。
- 收藏类目偏好饼图。
- 购买类目偏好饼图。
- 价位偏好饼图。

## 6. 智能推荐模块

### 6.1 模块目标

根据用户行为和商品特征生成推荐商品，并展示推荐理由。

### 6.2 推荐因子

| 因子 | 权重说明 |
|---|---|
| 类目偏好 | 浏览、收藏、加购、购买越多，权重越高 |
| 价位匹配 | 商品价格接近用户常看/常买价位 |
| 商品热度 | 销量、点击、加购较高的商品加分 |
| 新鲜度 | 用户未购买过的商品加分 |
| 评分 | 高评分商品加分 |

### 6.3 推荐输出

每个用户推荐 10 个商品，包含：

- 商品 ID
- 商品名称
- 商品图片
- 价格
- 推荐分
- 推荐原因

## 7. 预警模块

### 7.1 模块目标

发现低活跃、流失风险、转化异常和行为数据异常。

### 7.2 预警类型

- 低活跃提醒
- 流失风险预警
- 转化异常预警
- 行为数据异常预警

### 7.3 状态流转

```text
new -> acknowledged -> processing -> resolved
```

## 9. 权限与审计模块

### 9.1 权限原则

- 前端按角色显示菜单。
- 后端按角色校验接口权限。
- 敏感数据默认脱敏。
- 导出、查询、重置、预警处理均记录审计日志。

### 9.2 审计日志字段

- 操作者
- 角色
- 操作类型
- 资源类型
- 资源 ID
- 操作结果
- 失败原因
- 操作时间
